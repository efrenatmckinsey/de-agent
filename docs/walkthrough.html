Environment Setup

Step-by-step local development bootstrap

▶ Step 1: Clone & Install Dependencies 
# Clone and create virtual environment
git clone https://github.com/efrenatmckinsey/de-agent.git
cd de-agent
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# Copy environment template and fill in values
cp .env.template .env
# Edit .env with your API keys (optional for public APIs)
▶ Step 2: Platform Configuration (config/platform.yaml) 
# Central control plane: routing, scaling, storage, governance
platform:
  name: hydra-ingest
  environment: dev          # dev → local queue; prod → Kafka

topic_router:
  kafka:
    local:
      bootstrap_servers: "localhost:9092"
  topic: hydra.ingestion.commands

autoscaler:
  min_replicas: 1
  max_replicas: 20
  idle_shutdown_minutes: 45  # scale to zero when idle

storage:
  s3:
    bucket: hydra-data-lake
    base_paths:
      raw: s3://hydra-data-lake/raw
      curated: s3://hydra-data-lake/curated
  iceberg:
    catalog:
      type: glue             # AWS Glue catalog for Iceberg tables
▶ Step 3: Define Sources (config/sources/*.yaml)

    value_from_env: USDA_API_KEY

    requests_per_second: 5

schedule:
  cron: "0 2 * * *"       # daily at 2am

  classification: public
  retention_days: 365

Business Capabilities

🎯 Self-Optimizing Compute

Automatically selects Spark for large datasets and lightweight Python for small ones. Scales to zero when idle — you only pay for what you use.

🔍 Source Intelligence

Before ingesting, the platform probes each source to learn its schema, estimate volume, and detect if data has changed. No wasted processing on stale data.

🛡️ Built-In Governance

Every record is scanned for PII (emails, SSNs, credit cards). Data lineage tracks where each row came from and where it went. Quality gates prevent bad data from reaching your lake.

📊 Full Observability

Prometheus metrics, OpenTelemetry traces, and structured JSON logs — out of the box. Dashboards in Grafana show ingestion rates, error rates, and quality scores in real time.

Component Map

Every module, its responsibility, and source file

● Topic Router
Kafka/local queue routing with priority ordering. Commands: ingest, transform, backfill.
src/core/topic_router.py
● Context Layer
HTTP probing, schema inference from samples, volume estimation, freshness checks. TTL cache.
src/core/context_layer.py
● API Source
REST client with rate limiting, pagination (offset/cursor), retry with backoff on 429/5xx.
src/ingest/api_source.py
● Stream Source
SSE + simulated polling with high-water mark dedup and reconnect logic.
src/ingest/stream_source.py
● Batch Source
Full-dataset download with streaming to disk. JSON, CSV, Parquet formats.
src/ingest/batch_source.py
● DSL Parser
YAML → TransformSpec with discriminated union steps. Validates and recommends engine.
src/transform/dsl_parser.py
● Spark Engine
PySpark session with Iceberg extensions. Executes filter/flatten/agg/derive/join steps.
src/transform/spark_engine.py
● Python Engine
Pure-Python transforms with SQL expression evaluator (incl. CASE WHEN). Zero external deps.
src/transform/python_engine.py
● Iceberg Writer
S3/local Parquet output with PyIceberg catalog integration. Schema auto-detection.
src/storage/iceberg_writer.py
● PII Classifier
Regex + column-name heuristics for 8 PII types. Auto-escalates public→confidential.
src/governance/classifier.py
● Lineage Tracker
OpenLineage-compatible START/COMPLETE/FAIL events with schema + quality facets.
src/governance/lineage.py
● Quality Engine
8 check types: not_null, valid_range, unique_key, row_count, regex, freshness, referential, custom_sql.
src/governance/quality.py
● AutoScaler
CPU/memory/queue thresholds → scale up/down/shutdown. Local: multiprocessing. AWS: ECS/EKS.
src/core/autoscaler.py
● Orchestrator
Main DAG: consume commands → discover → ingest → transform → store → lineage → quality → metrics.
src/core/orchestrator.py

Governance in Action

Every ingestion cycle automatically performs these governance checks:

🔒 PII Detection

Scans every column for sensitive data patterns. If PII is found in a "public" dataset, the classification is automatically escalated and the team is alerted.

📈 Quality Gates

Configurable checks run before data lands in the lake. If critical violations are found, the pipeline halts — bad data never reaches downstream consumers.

🗺️ Data Lineage

Every row is traceable: where it came from (source API), what transforms were applied, and where it was stored. Full audit trail for compliance.

📉 Auto-Scaling Economics

Workers scale up during peak ingestion and shut down after 45 minutes of idle time. Compute costs track actual usage, not provisioned capacity.

1. Command Dispatch

# A command arrives on the Kafka topic (or local queue in dev):
{
  "source_id": "usda_food_data",
  "action": "ingest",
  "priority": 1,
  "payload": {}
}

# The orchestrator deserializes it into an IngestionCommand (Pydantic model)
# and routes based on action: ingest → full cycle, transform → transform-only

2. Context Discovery

# ContextLayer probes the source endpoint before ingestion:

context = await context_layer.get_or_discover(source_id, source_config)

# 1. endpoint_probe()  → HEAD request, parse headers (content-type, pagination, etag)
# 2. schema_discovery() → GET sample, infer column names + types + nullability
# 3. volume_estimation() → Extract total from pagination or extrapolate from sample
# 4. freshness_check()  → Compare ETag/Last-Modified to skip unchanged data

# Result is cached with TTL (default 1h) to avoid hammering the API
# Returns: SourceContext(inferred_schema, estimated_rows, content_type, ...)

3. Adaptive Ingestion

# The ingestor is selected based on source type in YAML config:
ingestor = select_ingestor(source_config["type"])  # api | stream | batch

# API:    httpx + rate limiter (semaphore) + pagination + retry (exp backoff)
# Stream: SSE or simulated poll with high-water mark dedup
# Batch:  Stream-to-disk download, parse JSON/CSV/Parquet

records = await ingestor.ingest(source_config=config, context=context)
raw_uri = await storage.write_raw(source_id=sid, records=records, context=context)

4. Engine Selection & Transform

# The DSL parser loads the transform spec from YAML:
spec = DSLParser.parse_file("config/transforms/food_enrichment.yaml")

# Engine auto-selection based on volume:
if spec.engine_hint == "auto":
    engine = "spark" if context.estimated_rows >= spec.volume_threshold else "python"

# Both engines execute the same step types:
#   filter → flatten → rename → cast → derive → aggregate → quality_gate → join

# Spark: pyspark DataFrame operations (df.filter, df.groupBy.agg, df.withColumn)
# Python: list[dict] with ExpressionEvaluator (supports CASE WHEN, IS NOT NULL, etc.)

5. Transform DSL Example

transform_id: food_nutrient_enrichment
engine_hint: auto
volume_threshold: 100000
source: usda_food_data

steps:
  - type: filter
    where: "nutrients IS NOT NULL"

  - type: flatten
    column: nutrients
    into: [nutrient_name, nutrient_value, unit]

  - type: aggregate
    group_by: [food_category]
    measures:
      - { column: nutrient_value, function: avg, alias: avg_nutrient }
      - { column: nutrient_value, function: max, alias: max_nutrient }

  - type: quality_gate
    checks:
      - { type: not_null, columns: [food_category, avg_nutrient] }
      - { type: valid_range, column: nutrient_density, min: 0, max: 10000 }

output:
  table: curated.food_nutrients
  format: iceberg
  partition_by: [food_category]

6. Governance Pipeline

# All three governance checks run on every ingestion cycle:

# A. PII Classification
result = classifier.classify_records(source_id, records, governance_config)
# Scans for: EMAIL, PHONE, SSN, CREDIT_CARD, IP, NAME, ADDRESS, DOB
# If PII found on "public" dataset → auto-escalate to CONFIDENTIAL

# B. Lineage (OpenLineage-compatible)
run_id = tracker.start_run("ingest:usda_food_data", inputs=[...], outputs=[...])
tracker.complete_run(run_id, outputs_with_facets=[...])
# Emits: schema facet, data quality facet, transform facet

# C. Quality Gate
qr = engine.run_checks(source_id, records, checks_config, gate_config)
# gate_config: { fail_on_critical: true, warn_threshold: 0.95 }
# If qr.passed_gate is False → pipeline halts, alert emitted

7. Scaling Behavior

# The autoscaler runs on a periodic loop (default: every 120s)

metrics = ComponentMetrics(
    cpu_percent=82.5,    # above 75% threshold → SCALE_UP
    memory_mb=4096.0,
    queue_depth=250,
    last_active=now,
)

decision = autoscaler.evaluate(metrics, config)  # → ScalingDecision.SCALE_UP
await autoscaler.apply_decision(decision)

# Local:  spawns multiprocessing.Process workers
# AWS:    calls ECS UpdateService (desired_count += 1)
# K8s:    HPA handles it natively via custom queue_depth metric
# Idle:   after 45min with no commands → SHUTDOWN (scale to zero)
